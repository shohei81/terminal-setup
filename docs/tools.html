Tools

Everything terminal-setup installs, and why.

A command is a word (or short phrase) you type in the terminal and press Enter to run. The examples below show what to type — lines starting with $ are commands you run, lines without are output or explanations.

Always installed

fzf zoxide yazi lazygit bottom
Always installed

These tools are installed automatically for everyone.

Ghostty

The terminal app itself — the window where you type commands. Ghostty is faster and looks better than the default Terminal.app that comes with macOS. terminal-setup configures it automatically with your chosen theme and font.

Open it like any Mac app — from Launchpad, Spotlight (Cmd+Space → type "Ghostty"), or your Dock.

github.com/ghostty-org/ghostty →

Hack Nerd Font

A font designed for the terminal. It includes thousands of small icons — folder symbols, file type icons, Git branch indicators — that other tools like Starship and lsd use to make their output cleaner. Applied automatically in Ghostty — nothing to configure.

github.com/ryanoasis/nerd-fonts →

Starship

The prompt is the line that appears before your cursor — it usually shows the current folder name. Starship makes the prompt more useful: it automatically shows which Git branch you're on, whether there are unsaved changes, which version of a programming language the project uses, and more. It loads silently every time you open a terminal.

# Your prompt looks something like this — no command needed:
~/projects/myapp git:(main ✗) ▸
github.com/starship/starship →

lsd

Type ls to see what files are in your current folder — like opening a Finder window, but as text. lsd makes that list much easier to read, with colors and icons for each file type. terminal-setup sets it up so ls automatically uses lsd.

$ ls # list files in the current folder
$ ll # list files with sizes, dates, and hidden files
$ lsd --tree # show folders and subfolders as a tree
github.com/lsd-rs/lsd →

bat

Type bat followed by a filename to read that file right in the terminal — without opening an editor. Code is shown with syntax highlighting (colors based on the programming language) and line numbers, making it much easier to skim.

$ bat README.md # read a file in the terminal
$ bat main.go # code shown with colors by language
github.com/sharkdp/bat →

fd

Use fd to search for files by name. Type a word and it finds every file with that name inside the current folder and all subfolders. Fast and simple.

$ fd config # find every file named "config"
$ fd .py # find every file ending in .py
$ fd .log ~/Library # search inside a specific folder
github.com/sharkdp/fd →

ripgrep

Use rg to search for a word or phrase inside files. Unlike fd which finds files by name, ripgrep looks at the contents. Type a word and it finds every file that contains it, showing the matching line. Also used behind the scenes by many AI tools and code editors.

$ rg "TODO" # find every file containing "TODO"
$ rg "api_key" ~/ # search your entire home folder
github.com/BurntSushi/ripgrep →

fastfetch

Displays a system info summary every time you open a new terminal — macOS version, CPU, memory, uptime, and more. You choose which items to show during setup.

$ fastfetch # show system info manually
github.com/fastfetch-cli/fastfetch →

zsh-autosuggestions

As you type, shows a faint ghost-text suggestion based on commands you've typed before. Press → (right arrow key) to accept the suggestion. The more you use the terminal, the better it gets at predicting what you're about to type.

# Type the start of a command you've run before:
$ git push origin main ← press → to accept
github.com/zsh-users/zsh-autosuggestions →

zsh-completions

When you press Tab while typing, the terminal tries to complete what you're writing — filling in a filename, folder, or command option. zsh-completions adds this Tab-completion support for hundreds of popular tools, so you can explore what a command can do without memorizing everything.

# Press Tab after typing a partial command:
$ git Tab # shows: add, commit, push, pull …
github.com/zsh-users/zsh-completions →
Optional — your choice

Powerful tools with a learning curve. Install what sounds useful — you can add more later.

fzf

fzf lets you search through a list by typing just a few letters — it narrows down the options as you type. The most useful way to use it: press Ctrl+R to search everything you've ever typed in the terminal. Find and re-run any past command without retyping it.

$ fzf # interactively filter any list
Ctrl+R # fuzzy-search command history
$ ls | fzf # filter file list and select one
github.com/junegunn/fzf →

zoxide

In the terminal, you move between folders by typing cd foldername. zoxide remembers which folders you visit most often, so instead of typing the full path you can type z and just a few letters — it figures out where you mean.

$ z proj # jumps to ~/projects/myapp
$ z down # jumps to ~/Downloads
$ zi # pick from recently visited folders
github.com/ajeetdsouza/zoxide →

yazi

A file browser that runs inside the terminal. Works like Finder, but with keyboard navigation — browse folders, preview files, and open or move them without leaving the terminal.

$ yazi # open the file browser
# arrow keys to navigate · Enter to open · q to quit
github.com/sxyazi/yazi →

lazygit

Git is a tool that tracks changes to files over time — commonly used for code. lazygit gives Git a visual interface inside the terminal: see what changed, save a snapshot (commit), and upload to GitHub, all without memorizing Git commands.

$ lazygit # open inside a folder tracked by Git
# space to stage · c to commit · P to push · q to quit
github.com/jesseduffield/lazygit →

bottom

Shows a live dashboard of how your Mac is performing — CPU, memory, running apps, disk, and network — all in the terminal. Like the Activity Monitor app that comes with macOS, but keyboard-driven and more detailed.

$ btm # open system monitor
# q to quit · arrow keys to navigate
github.com/ClementTsang/bottom →
